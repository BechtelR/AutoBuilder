# Phase 14 Pre-Work: Workflow Eval Suite & Context Optimization

*Created: 2026-04-13 | Source: Phase 7b shaping conversation*

## Vision

The high-performance of workflows is the ENTIRE utility of AutoBuilder. The eval suite is the mechanism by which workflows self-improve and validate themselves. Phase 7b builds a solid pre-activation dry run; Phase 14 elevates it to a full self-improving certification pipeline.

## Workflow Eval Suite

### Core Concept
A reusable, self-improving workflow certification framework that validates workflows before activation AND accumulates regression tests from production failures.

### Capabilities
- **Multi-pass evaluation**: Happy path, error paths, edge cases — multiple dry runs with increasing complexity
- **Lifecycle testing**: Pause/resume at checkpoint boundaries, mid-execution edit injection, escalation handling
- **Full E2E simulation**: Real LLM calls (lightweight model default, production model optional), real state flow, real gate evaluation
- **Pluggable test scenarios**: Workflow authors define their own synthetic inputs and expected outcomes
- **Machine-readable eval reports**: Structured output that feeds back into the improvement loop
- **Regression accumulation**: Production failures automatically generate new eval test cases
- **Self-improvement loop**: Workflow runs in production -> failures captured -> eval suite gets new test cases -> next version runs against expanded suite -> fewer production failures

### The Self-Improvement Feedback Loop
```
Workflow authored -> Eval suite certifies -> Activated -> Production execution
     ^                                                         |
     |                                                         v
     +-- Improvement proposal <-- Failure patterns captured <--+
              |
              v
     Eval suite expanded with new test cases from failures
```

### Relationship to Phase 7b Dry Run
Phase 7b implements the foundation: full E2E dry run with lightweight LLM, multiple passes, pause/resume/edit testing. Phase 14 wraps this foundation in the self-improving framework with regression accumulation and structured eval reporting.

## Context Optimizer

### Core Concept
Optimize the context assembly (instruction fragments, skills, memory, prompts) for each agent invocation to maximize output quality within token budgets.

### What It Optimizes
- Instruction fragment ordering and pruning (which fragments matter most for this task?)
- Skill selection refinement (which skills actually improve output for this deliverable type?)
- Memory relevance scoring (which memories are worth the tokens?)
- Prompt compression (can the node prompt be tightened without losing intent?)
- Model routing (which model performs best for this specific task profile?)

### Evidence-Based Optimization
- Track output quality signals per context configuration
- A/B test context variations across eval runs
- Accumulate performance data: "for node type X, including skill Y improved output quality by Z%"
- Feed back into skill matching, memory loading, and instruction assembly
