| 50 | InstructionAssembler — fragment-based prompt composition | 6 typed fragments (safety, identity, governance, project, task, skill) via InstructionAssembler; auditability via source tracking; enables Director governance without prompt rewriting. | 53 | System Reminders — ephemeral conversation nudges | Soft nudges (token budget, state changes) injected as tagged messages via before_model_callback; NOT for hard governance (governance in assembled system prompt); ephemeral by design — lost during recreation is acceptable. | 54 | Declarative agent definition files | Agent definitions are markdown files with YAML frontmatter (not Python dataclasses). Full replacement by name. Evolves Decision #51 — AgentRegistry scans files instead of receiving Python registrations. | 55 | SAFETY fragment formalized (extends #50) | Formalizes the non-overridable safety floor in InstructionAssembler: no data exfiltration, tool boundary enforcement, escalation protocol. | 56 | Hybrid CustomAgent model formalized (extends #12) | Formalizes the always-intended spectrum: purely deterministic (SkillLoaderAgent, LinterAgent) or hybrid (DependencyResolverAgent, DiagnosticsAgent — deterministic `_run_async_impl` with internal LiteLLM calls). `type` field is always `llm` or `custom`; hybrid is a behavioral subcategory, not a third type. | 57 | MemoryLoaderAgent formalized (extends #19) | Formalizes memory context loading as a deterministic CustomAgent in the pipeline (not an LLM-discretionary tool). | 58 | Agent security model formalized (extends #6, #33) | Formalizes the always-intended security boundaries: (1) Project-scope definition files restricted to `type: llm` only, `tool_role` ceiling validated against workflow manifest.
