# Data Model Patterns

This skill covers the conventions for defining SQLAlchemy 2.0 mapped classes and Pydantic models in AutoBuilder, and maintaining the type safety chain from database to TypeScript client.

## Type Safety Chain

AutoBuilder maintains a one-directional type flow:

```
SQLAlchemy mapped class → Pydantic response model → OpenAPI schema → generated TypeScript types
```

Never short-circuit this chain. Database objects never cross the gateway boundary. The Pydantic layer is the contract.

## SQLAlchemy Mapped Classes

Use declarative mapped classes with full type annotations (SQLAlchemy 2.0 style):

```python
from datetime import datetime, timezone
from sqlalchemy import String, DateTime, ForeignKey, func
from sqlalchemy.orm import Mapped, mapped_column, relationship
from app.models.base import Base

class Project(Base):
    __tablename__ = "projects"

    id: Mapped[str] = mapped_column(String(36), primary_key=True)
    name: Mapped[str] = mapped_column(String(255), nullable=False, index=True)
    description: Mapped[str | None] = mapped_column(String(2000), nullable=True)
    status: Mapped[ProjectStatus] = mapped_column(String(32), nullable=False)
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True), server_default=func.now(), nullable=False
    )
    updated_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True), server_default=func.now(), onupdate=func.now(), nullable=False
    )

    deliverables: Mapped[list["Deliverable"]] = relationship(back_populates="project")
```

Key conventions:
- `Mapped[T]` annotations required — enables pyright strict mode
- `Mapped[str | None]` for nullable columns (not `Optional[str]`)
- String enums stored as `String(N)` — never SQLAlchemy `Enum` type (migration pain)
- Timestamps: always `timezone=True`, use `server_default=func.now()`
- Primary keys: UUID strings (`String(36)`), generated by application layer before insert
- No `Any` — use explicit mapped types

## Enum Storage

Store enums as strings. Use a `TypeDecorator` only when the database type differs from the enum value. Direct string storage is preferred:

```python
# Enum value stored verbatim in DB column
status: Mapped[WorkflowStatus] = mapped_column(String(32), nullable=False)
```

Enums live in `app/models/enums.py`. Values must match names (uppercase):

```python
class ProjectStatus(str, enum.Enum):
    ACTIVE = "ACTIVE"
    ARCHIVED = "ARCHIVED"
```

## Relationship Patterns

Prefer explicit `relationship()` with `back_populates`. Use `lazy="raise"` in production to prevent N+1 queries from accidental sync access:

```python
# Parent side
deliverables: Mapped[list["Deliverable"]] = relationship(
    back_populates="project",
    cascade="all, delete-orphan",
    lazy="raise",
)

# Child side
project: Mapped["Project"] = relationship(back_populates="deliverables", lazy="raise")
project_id: Mapped[str] = mapped_column(ForeignKey("projects.id"), nullable=False, index=True)
```

For async loading, use `selectinload` or `joinedload` in queries — never rely on lazy loading in async context.

## Pydantic Response Models

Define in `app/gateway/models/`. Use `ConfigDict(from_attributes=True)` for ORM conversion:

```python
from pydantic import BaseModel, ConfigDict
from datetime import datetime

class ProjectResponse(BaseModel):
    model_config = ConfigDict(from_attributes=True)

    id: str
    name: str
    description: str | None = None
    status: ProjectStatus
    created_at: datetime
    updated_at: datetime
```

Convert with `.model_validate(db_obj, from_attributes=True)`. Never pass SQLAlchemy objects directly to callers.

Separate request and response models:
- `CreateProjectRequest` — inbound validation, no `id` or timestamps
- `UpdateProjectRequest` — partial fields, all optional
- `ProjectResponse` — outbound shape, what callers receive
- `ProjectListResponse` — collection wrapper with pagination metadata

## Pydantic Internal Models

For internal data transfer between layers (not API-facing), use `dataclass` or `TypedDict` instead of `BaseModel`. These carry no validation overhead and are appropriate for trusted internal data:

```python
from dataclasses import dataclass

@dataclass(frozen=True)
class ProjectSeed:
    """Internal struct passed from ARQ task to agent pipeline."""
    project_id: str
    name: str
```

## Field Definitions

Prefer simple field declarations without unnecessary `Field()`. Use `Field()` only when a constraint or default_factory is needed:

```python
# Simple — no Field() needed
name: str
description: str | None = None

# Field() for list defaults (pyright strict requirement)
tags: list[str] = Field(default_factory=lambda: list[str]())

# Field() for constraints
name: str = Field(min_length=1, max_length=255)
```

## Checklist

- All mapped columns have explicit `Mapped[T]` annotations
- Nullable columns use `Mapped[T | None]`, not `Optional[T]`
- String enums stored as `String(N)`, not DB-level enum
- Timestamps include `timezone=True`
- Relationships use `back_populates` and `lazy="raise"`
- Response models use `ConfigDict(from_attributes=True)`
- Request and response models are separate classes
- No `Any` types — use explicit mapped types or `TypedDict`
- Enums defined in `app/models/enums.py`, value equals name (uppercase)
